## üö® MOBILE-SPECIFIC FAILURE PATTERNS

### Common Mobile Issues to Watch For:

#### 1. Safe Area Violations
```
‚ùå Problem: Content hidden by notch/home indicator
‚úÖ Solution: Use SafeAreaView or useSafeAreaInsets()

Evidence:
[Screenshot showing content hidden by notch]
Recommendation: Wrap screen in SafeAreaView
```

#### 2. Android Back Button Issues
```
‚ùå Problem: Back button crashes app at root screen
‚úÖ Solution: Implement BackHandler to minimize app

Evidence:
[Crash log from Android back button]
Recommendation: Add BackHandler.addEventListener in root
```

#### 3. Keyboard Overlap
```
‚ùå Problem: Keyboard covers input field
‚úÖ Solution: Use KeyboardAvoidingView

Evidence:
[Screenshot showing keyboard hiding input]
Recommendation: Wrap form in KeyboardAvoidingView with behavior="padding"
```

#### 4. Performance Jank
```
‚ùå Problem: List scrolling drops to 30 FPS
‚úÖ Solution: Use FlatList with proper optimization

Evidence:
[Performance Monitor screenshot showing low FPS]
Recommendation: 
- Use FlatList instead of ScrollView with map
- Add getItemLayout for fixed height items
- Set removeClippedSubviews={true}
```

#### 5. SignalR Connection Failures
```
‚ùå Problem: WebSocket doesn't reconnect after backgrounding
‚úÖ Solution: Implement reconnection on AppState change

Evidence:
[Log showing connection lost on background]
Recommendation:
AppState.addEventListener('change', (state) => {
  if (state === 'active') reconnectSignalR()
})
```
